# Planning

Systemic task breakdown and implementation mapping.

## Core Guidelines

- **Plan First**: Write down a detailed roadmap in `implementation_plan.md` for any non-trivial modifications.
- **Checklists**: Create and update a `task.md` document tracking steps with `[ ]` (unstarted), `[/]` (in progress), and `[x]` (complete).
- **Seek Feedback**: Ask the user for approval on planning stages before execution.
