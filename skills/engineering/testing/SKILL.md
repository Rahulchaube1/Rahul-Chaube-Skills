# Testing

Unit tests, integration tests, E2E validation, and mock assertions.

## Core Guidelines

- **Isolation**: Mock external API calls and databases in unit tests.
- **Assertions**: Write deterministic, clear, and focused assertions.
- **Coverage**: Focus on testing complex business paths rather than simple getters/setters.
