# Prompt Engineering

Instruction tuning, XML formatting, constraints, and systemic prompts.

## Core Guidelines

- **Enclosures**: Separate inputs, instructions, and schemas using tags (e.g. `<system>`, `<data>`).
- **Few-Shot**: Provide concrete input/output examples for complex extraction or formatting tasks.
- **Negative Constraints**: Clearly specify forbidden actions (e.g., "Do NOT change variable names").
