# Best Practices

This guide outlines best practices for developers using or extending **Rahul-Chaube-Skills (RCS)** in their production AI pipelines.

---

## 🧑‍💻 Using RCS in Everyday Coding

### 1. Let the Agent Plan (Planning Mode)

When using Cursor Composer or Claude Code, encourage the agent to write a plan inside `implementation_plan.md` before writing code.

- Avoid instructing the agent to "just write the code."
- Instead, say: "Create a plan first, then wait for my approval."

### 2. Force Test-Driven Development (TDD)

When asking for changes, define the success criteria in terms of tests:

- "Implement feature X. First write a failing unit test, verify it fails, then write the code to make it pass."
- This forces the agent into a strict verification loop, reducing logic errors.

### 3. Review Diffs Surgically

When reviewing pull requests generated by an AI agent, verify that:

- No comments were deleted.
- No formatting style was altered (e.g. quote styles, indentation).
- No unrelated functions were refactored.

---

## 🛠️ Customizing and Writing New Skills

### 1. Write Clear Constraints

When defining prompts inside a `SKILL.md` file, use strict command language.

- **Good**: "You must only use relative imports."
- **Bad**: "Try to use relative imports if possible."

### 2. Include Examples

LLMs are highly responsive to few-shot learning. Provide a concise block of `Before` and `After` code or text representing the skill.

### 3. Keep Skills Focused

Do not create a skill like `full-stack-web-development`. Instead, split it into `frontend`, `backend`, and `state-management` skills. Focused instructions result in higher model alignment.
